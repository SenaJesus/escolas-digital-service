# escolas.digital — API

Este repositório contém a API REST que alimenta a interface do escolas.digital, uma plataforma web que reúne informações públicas sobre escolas brasileiras, permitindo que famílias pesquisem instituições por estado, cidade, bairro e nome, leiam avaliações e consultem dados oficiais de censo escolar, ENEM, IDEB e do Programa Dinheiro Direto na Escola. O projeto foi desenvolvido como trabalho da disciplina SCC0219 — Introdução ao Desenvolvimento Web do Instituto de Ciências Matemáticas e de Computação da Universidade de São Paulo.

A API foi construída em Node.js com TypeScript estrito, usando Express como servidor HTTP, Sequelize como ORM sobre PostgreSQL 16, zod para validação de entrada, JSON Web Tokens para autenticação no fluxo de avaliações, Nodemailer para envio de e-mails transacionais e Winston para logging estruturado. O código está organizado por feature dentro de src/features, com cada domínio (location, school, auth, review) seguindo uma separação clara entre domínio, infraestrutura e apresentação, de forma que as regras de negócio ficam isoladas dos detalhes de framework e banco de dados.

Para executar o projeto localmente é preciso ter Node.js 20 ou superior e Docker instalados. Depois de clonar o repositório, entre na pasta do projeto e instale as dependências executando npm install. Em seguida suba o banco de dados PostgreSQL com docker compose up -d, o que inicia um container chamado escolas-org-postgres escutando na porta 5433 da máquina local. Copie o arquivo .env.example para .env caso queira ajustar alguma variável, mas os valores padrão já funcionam para desenvolvimento.

Com o banco de pé, o passo seguinte é popular a base com os dados públicos que a API consome. O dump do banco de dados está versionado dentro do próprio repositório em db/dump/escolas_digital.dump, em formato custom comprimido do PostgreSQL. Para importá-lo, copie o arquivo para dentro do container executando docker cp db/dump/escolas_digital.dump escolas-org-postgres:/tmp/escolas_digital.dump e em seguida rode docker exec -i escolas-org-postgres pg_restore -U escolas -d escolas_org --no-owner --no-privileges /tmp/escolas_digital.dump. O processo leva alguns minutos porque restaura aproximadamente oito milhões de registros distribuídos entre estados, cidades, escolas, censos escolares, infraestrutura, acessibilidade, corpo docente, matrículas, cotas, registros educacionais e avaliações. Se preferir partir de uma base vazia e criar apenas a estrutura sem dados, rode npm run db:migrate, que aplica todas as migrations do Sequelize.

Depois de importar o dump, suba o servidor de desenvolvimento com npm run dev, que por padrão escuta em http://127.0.0.1:8001. O endpoint GET /health responde 200 quando a API está pronta para receber requisições. A interface web do escolas.digital espera encontrar a API exatamente nessa URL, mas ela pode ser alterada no front via variável de ambiente.

Para verificar que tudo está funcionando antes de enviar mudanças, é possível rodar npm run typecheck para checar a tipagem do TypeScript, npm run lint para validar o código com o ESLint e npm run test para executar a suíte de testes unitários com o Vitest. O comando npm run build compila o código para JavaScript dentro da pasta dist, e npm start executa essa versão compilada para simular o comportamento de produção.
